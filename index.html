1 Introduction

What we want to answer are two fundamental questions, using hardware basic operations (XOR, AND) as performance measure:

  • If the minimum amount of basic operations needed is x to build a secure n-bit permutation, how many do we need at least to build a secure 2n-bit permutation? (our guess, at least twice more)
  • If the minimum amount of basic operations needed is x to build a secure n-bit TBC with k-bit tweakey, how many do we need at least to build a secure n-bit TBC with 2k-bit tweakey? (our guess, much less than twice the number)

The paper will make an empirical analysis (not a formal one, that is probably impossible) to try to answer these questions. To strengthen our argument, we need to backup our performance and security models with actual numbers: take some of our model predictions and compare it with actual performance and security numbers.
